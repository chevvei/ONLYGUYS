# 旅行地图数据（可选扩展）

- **`travel-map.js`** 内嵌 **中国大陆 + 海南 + 台湾** 简化轮廓（WGS84），大陆主环刻意强化 **「雄鸡」示意** 国界；**世界** 为 **正射投影球形**（经纬网 + 旋转/视距交互）。**非测绘级**，仅作视觉示意。
- 若需 **测绘级国界 / 更拟真雄鸡**，可在此目录放置 GeoJSON，并在 `js/travel-map.js` 中改为 `fetch` 解析（需 `http(s)` 部署；`file://` 需本地服务）。
- 在线地图（高德、Mapbox、天地图等）需 **API Key**、域名白名单与合规说明，本静态演示默认 **不接** 瓦片服务。

## 「轮廓线」是什么？为什么不像课本里的「雄鸡」？

**轮廓线**在这里指的是：把国界 / 海岸线近似成一条 **闭合折线**（很多经纬度点依次连线），再在页面上画成 **一条有颜色的边**（有时连同内部填色）。你看到的是 **数学上的多边形边界**，不是照片，也不是精细地图软件里那种上万点的真实海岸线。

大家常说的 **「中国地图像雄鸡」**，多半是 **示意图、卡通或心理印象**（头在东北、尾在藏南、脚在海南等），笔画会 **夸张** 某些凹凸，让人一眼联想到鸡的形状。

本站为了 **零依赖、纯静态、文件要小**，用的是 **少量顶点** 拼出来的简化边界，所以：

- 形状会 **偏「不规则多边形」**，细节（半岛、海湾、岛屿锯齿）都被抹平了；
- **不会像** 教材插图那样刻意画成「一只完整的鸡」；
- 这是 **正常现象**，不是渲染坏了。

若你希望 **更接近真实国界轮廓**（仍不一定像卡通雄鸡，但会更像标准地图），需要换成 **官方或开放数据里的高精度国界 / 海岸线 GeoJSON**（顶点数量多很多），再由脚本生成路径——数据与审图合规需自行把关。
